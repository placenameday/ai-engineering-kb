# Spec-Driven Development (规格驱动开发)

> Survey date: 2026-03-05.

---

**Core idea:** Use well-crafted software requirement specifications as the primary artifact; AI agents generate code from specs. The paradigm shift: "intent is the source of truth" replaces "code is the source of truth."

**Key proponents:** Thoughtworks (published analysis 2025), GitHub (released Spec Kit open-source toolkit), Karpathy (fits within Software 3.0 vision).

**Two schools of thought:**
- **Radical:** Code is a disposable byproduct; specs are the sole source of truth
- **Pragmatic:** Specs drive generation, but executable code remains authoritative

**Human authority shift:** Responsibility moves from implementation to intent, policy, and ethics. SDD doesn't remove humans — it relocates their responsibility to the highest abstraction layers.

**Team structure inversion (Intent-Driven Development variant):** Traditional: 20% product / 60% engineering / 20% design. AI-era: 60% product judgment / 30% engineering architecture / 10% design precision.

**Why it matters:** Addresses "ambiguity moving at machine speed" — agents need constraints, not vague requirements.
