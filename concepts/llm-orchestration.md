# LLM Orchestration as Discipline (LLM编排学)

> Survey date: 2026-03-05.

---

**Core idea:** The coordination layer managing communication between LLMs, specialized agents, APIs, and databases. Components: task router, memory layer, tool hub, workflow engine, monitoring systems.

**Key capabilities:** prompt management, dynamic model selection, quality control, multi-model coordination, fallback logic, caching, batching, gating.

**Key frameworks (2025-2026):** LangGraph (graph-based task mapping), CrewAI 2.0 (multi-agent collaboration), Microsoft Agent Framework (combines Semantic Kernel + AutoGen), Flowise (visual/no-code).

**Why it matters:** Called "the make-or-break factor" for scaling beyond isolated AI pilots (ZDNet). The shift from individual model capabilities to systematic coordination of multiple components.
